## 2. Complete ElevenLabs Narration Script
**Word Count:** 292 words (Conversational, technical, and high-impact)
**Estimated Duration:** ~2m 10s to 2m 20s

> Hi everyone. Today, I'm excited to showcase the Hybrid Model Router, or HMR, which dynamically bridges local and remote LLMs.
> 
> In production, high-capacity remote models offer accuracy but accumulate massive token bills. Meanwhile, small local models running on Ollama are fast and free, but struggle on complex reasoning tasks.
> 
> Crucially, local models suffer from poor self-calibration. We cannot trust their self-reported confidence scores to decide when to escalate requests.
> 
> Conventional architectures statically route all traffic to one tier, wasting budget or sacrificing accuracy. HMR solves this by dynamically routing queries through an intelligent, multi-stage cascade.
> 
> Our pipeline is divided into three key areas: semantic and predictive Optimizers at the frontline, multi-signal Trust Evaluators, and a dynamic telemetry-driven Budget Adjuster.
> 
> Looking at the architecture flow, incoming queries check the semantic cache and predictive gate. Complex tasks bypass local models entirely, while standard tasks run locally before validation.
> 
> For example, a prompt asking for recursive Fibonacci code runs locally. While its syntax compiles and token entropy is low, it fails our semantic consistency check, triggering remote escalation.
> 
> This behavior is implemented in four core modules: the Unified Executor, the Trust Evaluator, the local Semantic Vector Cache, and the budget-aware threshold Adjuster.
> 
> HMR runs on Ollama locally and Fireworks AI remotely, wrapped in a FastAPI backend with Docker containment and a real-time React dashboard.
> 
> Our primary engineering innovations include statistical trust evaluation, embedding-based self-consistency checking, dynamic budget-to-burn adjustments, and predictive routing gate heuristics.
> 
> Moving forward, we plan to implement reinforcement learning for threshold tuning, batch-parallel consistency sampling, and device-quantized model calibration.
> 
> The code is fully open source. Thank you for listening!

## 3. Recording & Assembly Instructions

### Step A: Narration Generation (ElevenLabs)
1. Navigate to **ElevenLabs Speech Synthesis**.
2. Select a natural, professional voice (e.g., **Marcus**, **Adam**, or **Antoni**).
3. Adjust **Stability** to `65%` and **Clarity + Similarity** to `75%` to maintain a steady tone.
4. Paste the narration script above, generate, and download the `.mp3` file.

### Step B: Visuals Capture (Excalidraw / PPTX)
1. Open your presentation software using the updated presentation slides: **`updated_ppt.pptx`**.
2. Set up screen recording software (OBS Studio, QuickTime, or Loom) at `1080p` 60 FPS.
3. Record your screen while transitioning between slides at the exact timestamps matching the **Video Timeline** (from Slide 1 to Slide 12).

### Step C: Silent Live Demo Recording
1. Run the project locally (`make up` command to launch container services).
2. Record a browser window showcasing:
   - Submitting simple prompts that cache-hit or run on the local tier (fast response).
   - Submitting complex math/reasoning queries showing automatic predictive bypass or escalation to remote Llama-3 70B.
   - Showing the **Trust Signals** scorecard showing logprobs entropy and local consistency results.
   - Displaying the real-time cost-saving stats dashboard.
3. Trim this recording to exactly **40 seconds** of clean, non-narrated screen capture.

### Step D: CapCut Editing
1. Create a new CapCut project and import the voiceover track, the slides recording, and the demo clip.
2. Cut the slide transitions to sync perfectly with the voiceover boundaries in the timeline.
3. Append the 40-second silent demo video directly after Slide 12 (starting at `02:20`).
4. Export the completed video in `1080p` resolution at `30fps` or `60fps`.
